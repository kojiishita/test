概要

ドキュメントの目的

本ドキュメントは、KenpoWave21(以下KW21)の新規開発にあたり、システム全体が同一の水準の質を確保するために規定する。

本規約の適用範囲

KW21開発の設計者、画面デザイナ、プログラマ、アイコン作成者、テスタ、ヘルプ作成者およびマニュアル等のドキュメント作成者を本規約の対象者とする。

西暦和暦変換処理について

西暦から和暦、和暦から西暦への変換を行う際は、サーバ、クライアントモジュールともにプロジェクト「KCDATECL\_V2」から生成されるDLLを使用する。サーバモジュール用とクライアントモジュール用とでファイル名が異なる。

サーバ用 → KCDATECL\_V2.dll

クライアント用 → KCDATEVB\_V2.dll

各APから、それぞれのDLLを呼び出し、変換処理を行うこと。

クライアントの場合は、ローカルから直接、「KCDATEVB\_V2.dll」の関数を呼び出すのではなく、「C\_Date.vb」の関数を介して呼び出すこと。

マイクロソフトが提供する標準関数(JapaneseCalendar Class)を使用したり、ローカルで変換関数を用意し使用することを禁ずる。

また、画面の和暦の入出力については、「UbsKenpo.dll」の「KenpoDate」コントロールを使用すること。

入出力ファイルを取り扱うメニュについて

入出力ファイルを取り扱うメニュにおいては、以下の2点を考慮した上で設計を行うこと。

- 複数のメニュで、1つのファイルを用いた処理を行う設計にしない。
- 一つのメニュで、1つのファイルを複数回に亘り、参照や取込を行う設計にしない。
- 制限理由
セキュリティ対応として、入力ファイル・中間ファイルの後処理を設けたことにより、1つのファイルに対して、参照や取込を何度も行うことが、仕様として適切では無くなったため。

●既存のメニュにおいて問題となった例

【データ交換編集プログラム】にて、KENPO2.csvをもとに“所属・配付マスターの生成”と“所属・配付・社員のセット”の２つの変換を行う仕様であるが、それぞれの変換処理を2回に分けて実行したため、“所属・配付マスターの生成”変換後に、ファイル後処理により、KENPO2.csvが削除されてしまい、“所属・配付・社員のセット”ができなくなった。

“所属・配付マスターの生成”と“所属・配付・社員のセット”を1度に実施する仕様ならば、本件の問題は発生していなかった。

KWLOCATION.iniに画面位置情報を保持するメニュについて

“kw21\\client\\kw21sys\\KWLOCATION.ini”に画面位置情報を保持するメニュについて、画面が閉じられた際、KWLOCATION.iniに更新対象のセクションがなければ、プログラム側でセクションや必要なキーをファイルの最下部に、追加すること。

クエリにおける判定処理について

- 実行する人が、実行する処理を（@SHORIKBN）指定できるようにする。

- 該当の処理区分を選択しなかった場合は、クエリ内を実行しないようにする。
例）DECLARE @SHORIKBN char(1)

-- 1：画面表示　2：レコード削除

SET @SHORIKBN = '1'

IF @SHORIKBN != '1' and @SHORIKBN != '2'

BEGIN

SELECT @PRTSTR = '処理区分に、1、2以外が設定されています。'

PRINT @PRTSTR

GOTO End\_Label

END

クラス構造

KW21において、クラスの構造は以下の通りとする。

■ひとつのVBファイルに対して、ひとつのクラスを記述する。ふたつ以上のクラスを記述してはならない。

■VBファイル名とクラス名は、同一とする。

例）クラスの宣言が「Public Class SampleException」であれば、ファイル名は、“SampleException.vb”となる。

■各クラスの属する名前空間については、3.1.9クライアントモジュールの名前空間を参照すること。

■処理に必要なコーディングの記述は、ひとつのクラスに集約するのではなく、役割や処理内容ごとに細分化したクラスを用意し、そこに記述すること。

クラス名

先頭文字は大文字とし、複数の単語で構成される場合は、各単語の先頭を大文字にする。

先頭に接頭辞を付加しないこと。“\_”は使用禁止とする。

誤った例)  Public Class C\_SampleException

正しい例）Public Class SampleException

インターフェイス名

先頭文字は大文字とし、複数の単語で構成される場合は、各単語の先頭を大文字にする。

原則としてクラス名の先頭に“I”を付加したものを命名すること。

例）Public Interface ISampleException

構造体名

先頭文字は大文字とし、複数の単語で構成される場合は、各単語の先頭を大文字にする。

接頭辞を付加しないこと。“\_”は使用禁止とする。

**E****xcel****のテンプレートファイルをもとに、データを展開する場合**

Excelのテンプレートファイルは、拡張子を“xltx”とし、“kw21\\client\\kw21xls”に保存する。

Excelのテンプレートファイルをコピーして“xlsx”形式のファイルを作成後、作成されたファイルに値を出力しないこと。（ユーザが意図的に保存して作成したファイルではないファイルが作成されてしまうため。）

**データベース接続情報の取得**

データベース接続情報は、「VBAConnectcionInfo.dll」のGetKW21ConnectionStringメソッドを使用し、共有メモリに展開されたTOGO.ini.$uencsの設定内容を取得する。Kenpo.ini等の設定ファイルから直接、データベース接続情報を取得することを禁止する。

なお、RS21で本メソッドを使用する場合、「GetKW21ConnectionString」を「GetRS21ConnectionString」に変更すること。
