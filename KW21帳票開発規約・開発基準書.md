概要

ドキュメントの目的

本ドキュメントは、KenpoWave21  (以下KW21)帳票プログラム開発にあたり、システム全体が同一水準を確保するために規定する。

本規約の適用範囲

KW21開発の設計者、帳票デザイナ、プログラマ、テスタ、およびマニュアル等のドキュメント作成者を本規約の対象者とする。

一般規約

レポート定義ファイルの形式について

セクションレポート形式（rpx）を使用すること。

rpxファイルの外部出力と参照

帳票レイアウトファイルはrpxファイルとして外部出力することとする。rpxファイルの保存場所はD:\\kw21\\client\\kw21ar\\rpxとし（kw21インストール先ドライブによって「D:\\」は変わる）、各帳票プログラムは必ずrpxファイル保存場所を参照すること。rpxの保存場所はkenpo.iniを参照する。

Kenpo.ini設定例）

メニュ起動時のレポート定義ファイル格納場所のチェック

帳票出力メニュ起動時にKenpo.iniよりレポート定義ファイルの格納場所の設定を読み取り、ディレクトリの存在チェックを行う。メニュ画面のFormLoad時にCheckRPX関数を呼び出すことで行い、格納場所ディレクトリが存在しない場合はメニュを終了する。

帳票ビューワの共通化

帳票の印刷や画面出力を行う際に必要な帳票ビューワは帳票毎の作成では無く、共通化し外部プログラムとして起動する。ファイル名は「ARVIEWER.exe」とし、保存場所はD:\\kw21\\client\\kw21sys(ドライブはインストール先により変わる)とする。APと連動して起動する機能と単独で起動できる機能を有する。

帳票プレビュ実行時の中間ファイル作成場所

AP連動起動時、KW21の画面の「印刷」ボタン押下後、中間ファイルとしてxmlファイルを作成する。xmlファイルの作成場所はkenpo.iniで定義することとする。記述箇所は「1System.ini」とする。

記述例）

帳票プレビュからのCSV出力処理用のコピーMDB作成場所

帳票プレビュより、CSV出力処理を行う為にAPで作成したMDBのコピーを行っているが、MDBの作成場所はkenpo.iniで定義することとする。記述箇所は「1System.ini」とする。

帳票ビューワウィンドウタイトルの出力

画面プレビュ時の帳票ビューワウィンドウタイトルは各帳票の名称を表示させることとする。帳票名称は各RPXファイルの計算フィールドのDefaultValueプロパティに記述することとし、帳票ビューワが参照することで名称を表示する。尚、その際Nameプロパティには「ReportName」と記述する（下記手順を参照）。制限事項として帳票の内容が異なっても、RPXファイルが同一だと、名称は常に同一のものとなる（例：証管理簿の発行簿と未回収簿はどちらも「証管理簿」という名称となる）。

- ランタイムデザイナを起動する。
- 右上のウィンドウの「フィールド」→「計算フィールド」にて右クリックから計算フィールドを追加する。
- ２）で追加した計算フィールドのNameプロパティを「ReportName」に変更し、DefaultValueプロパティに帳票名称を設定する。
命名規約

原則KW21開発規約・開発基準書に準ずる。以下ActiveReports固有のものについて記述する。

- レポート定義ファイル（RPXファイル）
「プログラムID下５桁」+SEQ２桁（01からカウントアップ）.rpxとする。ファイル名は大文字とし、拡張子は小文字を使用する。

例　決定通知書（決定分）出力　プログラムＩＤ：KCPAAB11　→   AAB1101.rpx

- 印章画像ファイル（TIFFファイル）
拡張子を除きRPXファイル名と同名とする。

例）

RPXファイル：AAB1101.rpx　→　TIFFファイル：AAB1101.tiff

- 印章位置定義ファイル（INIファイル）
拡張子を除きRPXファイル名と同名とする。

例）

RPXファイル：AAB1101.rpx　→　INIファイル：AAB1101.ini

プリンタ設定

印刷方向である「縦」もしくは「横」の指定をする。デザイン時の用紙サイズについては、横向き用紙はA3、縦向き用紙はA4を基本とする。

ただし、帳票レイアウトやフォントサイズによってはそれらのサイズで収まり切らない場合があり、その時は横向き用紙をユーザ定義サイズ、縦向き用紙をA3サイズでデザインを行う。

（印刷方向が「横」の場合、目安としてフォントサイズが9Ptの場合メイリオ全角131文字、MS明朝全角130文字以上の際にA3サイズを超え、ユーザ定義サイズとなる。）

用紙サイズがA3の場合、初めに基本フォントサイズを12ptで設計し、収まりきらない場合は基本フォントサイズを9Ptに、それでも収まりきらない場合はユーザ定義サイズの用紙を使用する。

用紙サイズがA4の場合、以下の手順でレイアウトが収まるかを確認し、収まらない場合は用紙サイズを変更して対応する。

- 用紙サイズをA4、フォントサイズを9ptで設計
- 用紙サイズをA3、フォントサイズを12ptで設計
- 用紙サイズをA3、フォントサイズを9ptで設計
④用紙サイズをユーザ定義サイズ（420mm×594mm）、フォントサイズを17.25ptで設計

※縦・横は設計する帳票に応じて変更すること

また専用帳票の作成時は都度ユーザ定義サイズを指定すること。その際、原則、縮刷率100％（等倍）でデザインする。

帳票セクション規約

ヘッダ

帳票の要件に応じて、レポートヘッダ、ページヘッダ、グループヘッダを使い分ける。

フッタ

帳票の要件に応じて、レポートフッタ、ページフッタ、グループフッタを使い分ける。

セクションのKeepTogetherプロパティ

セクションの途中で改ページが発生するのを防ぐため、グループヘッダ／フッタ、ディティールのKeepTogetherプロパティの値はTrueとすること。

帳票形式によるセクションの使い分け

- 表紙を付ける場合
表紙のデザインはレポートヘッダにて行うこと。

- 単一形式（各ページのデザインが同じ）
全てのデザインをディティールセクションにて行うこと。

- 一覧表形式
各ページの表タイトルはページヘッダにデザインする。

表の列名、支部名称、ページ数、印刷日時は、グループヘッダにデザインする。

小計や総合計は、グループフッタにデザインし、集計単位の制御はグループヘッダのDataFieldプロパティにて行う。

明細部はディティールにデザインする。

各ページの表下部に出力したい項目がある場合は、ページフッタにデザインする。

合計行の制御

合計行の制御を行う場合、グループヘッダのDataFieldプロパティに集計単位の項目名をセットし、出力制御を行うこととする。

例）一覧表形式の帳票で事業所、支部ごとに合計行を出力したい場合

帳票デザインは「支部ヘッダ、事業所ヘッダ、明細部、事業所フッタ、支部フッタ」

の順番にする。支部ヘッダと事業所ヘッダのDataFieldにそれぞれ、支部コードと事業

所コードをセットすることで、それぞれの値が変わったタイミングで、合計行を出力す

るよう制御できる。

Datafieldプロパティ設定例）

項目の最大桁数

各出力項目の最大桁数は各項目が下記に示す文字が出力できるよう、設計することとする。メイリオフォントの場合、半角文字はプロポーショナルフォントとなる為、半角2文字=全角1文字とは常にならないので注意すること。

最大桁数分の出力幅が確保されているかデザイン時に確認できるようテキストボックスのTextプロパティに以下の表の文字を記述しておくこと。

| 出力項目 | 文字 |
| --- | --- |
| 全角項目 | Ｋ（全角大文字） |
| 数値 | 9999（カンマなし）、9,999（カンマあり） |
| 日付 | W99.99.99（※1）、99-99-99 |
| 口座名義 | X(半角大文字) |
| 半角カナ | ｱ(半角カナ) |
| 半角英数混在項目（アルファベット含む）<br>例：所属コード、配付コード | W(半角大文字) ※2 |
| 全角・半角混在項目<br>例：住所、備考、相手先 | Ｋ（全角大文字）※3 |

※1テキストボックス内の文字位置を右詰とすること。先頭の半角アルファベットにより出力位置にずれが生じるため。

※2 出来る限り、出力項目を拡げることで、最大桁数を維持すること。

※3 K（全角大文字）でデザインし、まだ広げられる余裕があるようであれば、主管部まで問い合わせること。（半角2文字が、全角１文字よりも幅を多くとるため）

それぞれ出力最大桁数をセットしておくこと。

コントロールの数

コントロール数が増加するに従い、メモリ使用量が増加する傾向があり、パフォーマンスに影響が出る為、コントロールは可能な限り数を少なくしてデザイン設計すること。

コントロールのプロパティ設定

- WrapModeプロパティ
出力する文字列の長さがテキストボックスの幅を超えた場合、誤った箇所で区切られて改行が発生するため、WrapModeプロパティは「NoWrap」に設定する。

- CharacterSpacingプロパティ
出力文字列が以下の条件に該当し、出力する文字列の長さがテキストボックスの幅を超えた場合、誤った箇所で区切られて改行が発生する為、CharacterSpacingプロパティは「0」に設定する。

- 全角が出力される項目で、半角アルファベット及び半角数字が混在する項目。
- 半角カナが出力される項目で、半角アルファベット及び半角数字が混在する項目。
（1）、（2）の現象について以下に例を示す。

- MultiLine = True, WrapMode = NoWrap, CharacterSpacing = 0（規約通りの例）
WordWrap=Falseの為、テキストボックスの幅を超えた文字列を出力しても、自動改行は行われない（正常な動作）。

- MultiLine = True, WrapMode = WordWrap, CharacterSpacing = 0

- MultiLine = True, WrapMode = NoWrap, CharacterSpacing = 1

印章出力機能

帳票出力時、一部の帳票について、健康保険組合印（印章）を出力可能とする。

処理概要

印章出力処理は以下のとおり。

- KenpoWave21帳票ビューワがRPXファイル内の印章出力用スクリプトを読み込む。
- RPXファイルのスクリプトにて印章位置定義ファイルを読込み、Pictureコントロールの各プロパティに反映させる。
- RPXファイルのデザインに配置されているPictureコントロールに、印章イメージを出力する。
印章画像ファイルの仕様

- ファイル形式はTIFF形式とする。
- 印章出力の対象帳票は、TIFFファイルを常に参照することとする。その為、印章出力を行わない場合については白紙（透明）のTIFFファイルを参照し、出力処理を行う。
印章の出力位置は、印章位置定義ファイルより取得する。

- 印章出力機能に使用するTIFFファイルのファイルサイズは100KB以下とする。
印章位置定義ファイルの仕様

- ファイル形式はINI形式とする。
- 印章出力対象のRPXファイル毎にINIファイルを作成する。
設定内容

セクション名は拡張子を除いたRPXファイル名とする。

セクションの説明となるコメントには、帳票名を記述する。

以下の4つのキーを設定する。全ての設定値は小数点第3位までの指定とする。

| キー | 説明 | AP初期出荷時の既定値 |
| --- | --- | --- |
| PosX | 印章の出力位置（X座標）を<br>設定する。（単位：cm） | 健保名を全角15文字として末尾の文字に<br>重ならない位置。 |
| PosY | 印章の出力位置（Y座標）を<br>設定する。（単位：cm） | 健保名（1行目）のコントロール下端とPictureコントロールの下端が揃う位置。 |
| Width | 印章の出力サイズ（幅）を<br>設定する。（単位：cm） | A4帳票：1.634<br>A3帳票：2.334 |
| Height | 印章の出力サイズ（高さ）を<br>設定する。（単位：cm） | A4帳票：1.634<br>A3帳票：2.334 |

帳票ビューワ

帳票ビューワは共通のものを使用するが、帳票によってCSV出力の可否がある為、「CSV出力」ボタンの表示・非表示の制御を行う。CSV出力の可否はprint.defのCSV出力フラグを参照する。CSV出力可能帳票をプレビュした時のみ「CSV出力」ボタンを帳票ビューワに表示する。

出力項目

CSVデータの出力項目は、MDB項目をそのままカンマ区切りで出力する。ただし、文字列型のデータ及び項目タイトルについては「"”」付で出力する。定義は帳票毎に持つこととする。

例）「SIBUCODE」→「”支部コード”」

明細の内容がコードや区分のみで、その内容をユーザが理解出来るようにする必要があるものについては、それぞれのコード・区分の内容を項目タイトルに記述する。

例）「KANJYOKBN」→「”勘定区分(2:一般勘定支出、4:介護勘定支出)”」

ただし、性別については、項目名部分を省略し、コードの内容のみ記述する。

例）「SEIBETU」→「” 5:男、6:女”」

帳票に出力していない項目の先頭に「■」を付ける。現在使用していない項目については「システム固定項

目」と出力する。（例：管理事業所コード等）

項目タイトルは以下の順序で命名すること。

- 帳票出力項目と１対１の項目は、帳票出力項目名とする。
- 帳票出力項目と１対１でない項目は、帳票出力項目名＋アルファとする。
- 帳票出力項目でなく、データベース項目と１対１の項目は、データベース項目名称とする。
- ①②③以外の項目は個別に項目名を検討する。
その他命名上の注意

- 同義の項目は、システム全体で項目名を統一する。
- カナは全角、英数字や記号（コロンなど）は半角とする。
- 区分やフラグはその内容も項目名に含む。
例）”5:男、6:女”や”勘定区分(2:一般勘定支出、4:介護勘定支出) ”など。
